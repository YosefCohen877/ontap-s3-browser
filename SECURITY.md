# Security Policy

## Reporting Security Vulnerabilities

If you discover a security vulnerability in this project, please report it responsibly.

**Do not** create public GitHub issues for security vulnerabilities.

Instead, please email security concerns to: [cyosef22@gmail.com]

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes

We will acknowledge receipt within 48 hours and provide a more detailed response within 7 days indicating our next steps.

## Security Considerations

- This application handles S3 credentials and data
- Always run behind HTTPS in production
- Use strong, unique credentials
- Regularly rotate access keys
- Monitor access logs for suspicious activity
